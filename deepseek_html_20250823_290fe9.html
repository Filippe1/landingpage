The firewall acts as a barrier, allowing legitimate traffic while blocking malicious requests and unauthorized access attempts.

Risks Without a Firewall

  • Unauthorized access to your server
  • Brute force attacks on login services
  • DDoS attacks that overwhelm your server
  • Exploitation of vulnerabilities in open ports
  • Malware installation and system compromise
  • Data theft or destruction

Benefits of a Firewall

  • Controls incoming and outgoing network traffic
  • Blocks unauthorized access while allowing legitimate traffic
  • Prevents brute force attacks by limiting connection attempts
  • Filters traffic based on predefined security rules
  • Provides logging of suspicious activities
  • Adds an essential layer to your security strategy

How to Set Up UFW on Ubuntu

UFW (Uncomplicated Firewall) is a user-friendly interface for managing iptables firewall rules.

Installation and Basic Setup

# Install UFW (usually pre-installed on Ubuntu)
sudo apt install ufw

# Deny all incoming connections by default
sudo ufw default deny incoming

# Allow all outgoing connections by default
sudo ufw default allow outgoing

Essential Firewall Rules

For a web server, you need to allow specific ports:

# Allow SSH connections (change port if needed)
sudo ufw allow 22/tcp

# Allow HTTP and HTTPS traffic
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# Enable the firewall
sudo ufw enable

# Check firewall status
sudo ufw status verbose

Without Firewall

  • All ports are accessible from anywhere
  • Vulnerable to port scanning and attacks
  • No protection against brute force attacks
  • Malicious traffic can reach your services directly
  • Server logs filled with attack attempts

With Firewall

  • Only necessary ports are open (HTTP, HTTPS, SSH)
  • All other ports are blocked by default
  • Protection against unauthorized access attempts
  • Cleaner server logs with only legitimate traffic
  • Added security layer against vulnerabilities

Advanced Firewall Configuration

For enhanced security, consider these additional measures:

Rate Limiting (Prevent Brute Force Attacks)

# Limit SSH connection attempts
sudo ufw limit ssh

# Alternatively, if you changed SSH port:
sudo ufw limit 2222/tcp

Specific IP Restrictions

# Allow SSH only from your IP address
sudo ufw allow from 192.168.1.100 to any port 22

# Allow a specific network range
sudo ufw allow from 192.168.1.0/24 to any port 22

Application-Based Profiles

# View available application profiles
sudo ufw app list

# Allow Nginx (opens both port 80 and 443)
sudo ufw allow 'Nginx Full'

Testing Your Firewall Configuration

After setting up your firewall, verify it's working correctly:

# Check which ports are open
sudo ufw status

# Check listening ports on your server
sudo netstat -tulpn

# Test from another machine (replace with your server IP)
nmap -sS 192.168.1.100

Use online tools like https://www.yougetsignal.com/tools/open-ports/ to verify that only ports 80 and 443 are open from the internet.

Maintenance and Monitoring

A firewall requires ongoing maintenance:

  • Regularly review and update firewall rules
  • Monitor firewall logs for suspicious activity
  • Keep your system and firewall software updated
  • Regularly audit open ports and services
  • Implement additional security measures like fail2ban
# Check firewall logs
sudo less /var/log/ufw.log

# Monitor live firewall activity
sudo tail -f /var/log/ufw.log
